# Travel Telegram Bot v2

Обновлённый Telegram-бот для планирования поездок и координации группы.

## Что изменилось

Первая версия была ботом-координатором: статусы участников, даты, заметки и сводка.

Теперь ядро продукта другое: бот сначала помогает **спроектировать саму поездку**, а уже потом помогает группе **согласовать участие**.

### Новый фокус
- принимает запрос в свободной форме;
- собирает travel-brief;
- предлагает маршрут по дням;
- даёт грубый бюджетный ориентир;
- подсказывает по логистике и формату проживания;
- предлагает альтернативные направления;
- сохраняет групповые функции: статусы, голосование по датам, сводка.

## Что умеет эта версия

### Планирование поездки
- `/plan <текст>` — создать поездку из свободного описания;
- `/newtrip` — пошаговый мастер создания поездки;
- `/brief` — показать структуру поездки;
- `/itinerary` — показать маршрут по дням;
- `/budget` — показать бюджетный ориентир;
- `/route` — показать логику дороги и перемещений;
- `/stay` — показать рекомендации по проживанию;
- `/alternatives` — показать альтернативные направления.

### Координация группы
- `/summary` — показать итоговую сводку;
- `/status` — отметить участие;
- `/participants` — показать список участников;
- `/adddate 12–16 июня` — добавить вариант дат и запустить голосование;
- `/settings` — включить / выключить напоминания;
- `/archive_trip` — закрыть активную поездку.

### Быстрые правки
- `/budget комфорт` — сменить бюджетный сценарий и пересчитать ориентир;
- `/setdestination Владивосток` — сменить направление;
- `/setdates 12–16 июня` — обновить даты;
- `/interests природа, еда, история` — обновить интересы и пересобрать маршрут;
- `/notes купить билеты до пятницы` — обновить открытые вопросы.

## Пример запроса

```text
/plan Хочу поехать с друзьями на 5 дней во Владивосток, нас 4, из Новосибирска, бюджет средний, любим море, еду и красивые виды
```

## Важно про данные

Эта версия **не использует live-цены** и **не подключена к билетным / hotel API**.

Что это значит:
- маршруты и бюджет — это встроенная эвристика;
- бот даёт полезный черновик поездки, а не финальную бронь;
- бюджет — ориентировочный диапазон, а не рыночная цена в моменте.

### Основные модули
- `travel_planner.py` — парсинг travel-запроса и генерация плана;
- `database.py` — SQLite-хранилище поездок, участников, дат и настроек;
- `bot/handlers.py` — команды Telegram и сценарии диалога;
- `app.py` — регистрация команд и запуск polling.

## Быстрый запуск

1. Создайте бота через `@BotFather` и получите токен.
2. Скопируйте `.env.example` в `.env`.
3. Подставьте токен в `TELEGRAM_BOT_TOKEN`.
4. Установите зависимости:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

5. Запустите:

```bash
python app.py
```

## Деплой на Render.com (Git Provider)

1. Установите Git на Windows: скачайте **Git for Windows** с `https://git-scm.com/download/win` и установите (подойдёт дефолтная установка).
2. Создайте репозиторий на GitHub и запушьте код.
3. В Render выберите **Git Provider**, подключите репозиторий и создайте сервис.
4. В Render задайте переменные окружения (Secrets):
   - `TELEGRAM_BOT_TOKEN`
   - `OPENROUTER_API_KEY` (если используете `/planai`)
   - `OPENROUTER_MODEL` (например `stepfun/step-3.5-flash:free`)
   - `DATABASE_PATH=/var/data/travel_bot.db`
5. Включите **Persistent Disk** с mount path `/var/data`, чтобы SQLite не терялась при рестартах.
6. Если Render не дал доступ к `/var/data`, бот автоматически переключится на локальный файл `travel_telegram_bot/data/travel_bot.db` и не упадёт на старте. В этом режиме база будет временной, пока не заработает Persistent Disk.

## Поведение в групповых чатах

- бот работает и в личном чате, и в группе;
- для групп доступны статусы участников и голосование по датам;
- если позже захочется автoанализ обычных сообщений чата, в `@BotFather` нужно будет отключить privacy mode;
- в текущей версии бот работает по явным командам и не читает переписку автоматически.
