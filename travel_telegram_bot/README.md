# Travel Telegram Bot v2

Telegram-бот для совместного планирования поездок в группе. Бот помогает превратить обсуждение в чате в понятный travel-brief, сводку по участникам и рабочий черновик поездки.

## Что умеет MVP
### Основной сценарий
- добавляете бота в чат поездки;
- включаете авто-анализ в `/settings`;
- обсуждаете поездку обычными сообщениями;
- бот собирает черновик, показывает погоду и даёт русские ссылки на билеты и жильё;
- если нужны отдельные сценарии по жилью, используете `/hotels`;
- полный план всегда доступен через `/summary`.

### Дополнительные команды
- `/plan <текст>` — создать поездку из свободного описания
- `/newtrip` — пошаговый мастер создания поездки
- `/summary` — показать краткую сводку
- `/trips` — показать историю поездок в чате
- `/select_trip <id>` — вернуть поездку из истории в активный режим
- `/brief` — показать структуру поездки
- `/hotels` — быстрый сценарий по жилью и подготовка под реальные варианты через Playwright
- `/itinerary` — показать маршрут по дням
- `/budget` — показать или обновить бюджет
- `/route` — показать логистику
- `/stay` — показать рекомендации по проживанию
- `/alternatives` — показать альтернативные направления

### Координация группы
- `/status` — отметить статус участия
- `/participants` — показать статусы участников
- `/adddate 12–16 июня` — добавить вариант дат и голосование
- `/share` — поделиться текущим планом
- `/settings` — настроить напоминания и авто-черновики
- `/archive_trip` — перевести активную поездку в архив

### Быстрые правки
- `/setdestination Владивосток`
- `/setdates 12–16 июня`
- `/interests природа, еда, история`
- `/notes купить билеты до пятницы`
- inline-кнопка `Изменить` под summary

## Поддерживаемые сценарии
- друзья или коллеги обсуждают короткую поездку прямо в Telegram;
- организатор хочет быстро собрать черновик маршрута и бюджета;
- группе нужно договориться по составу участников и датам;
- в чате должна быть одна понятная активная поездка вместо десятков сообщений.

## Ограничения MVP
- бот не бронирует билеты и жильё;
- точных live-цен в самом боте пока нет, но в summary есть ссылки на реальные поиски билетов и жилья;
- одна активная поездка на чат;
- история архивных поездок сохраняется в базе, но основной UX сфокусирован на активной поездке;
- архивную поездку можно снова сделать активной через `/trips` и `/select_trip`;
- авто-анализ сообщений в группе можно отключить через `/settings`.

## Privacy mode и поведение в группах
- бот работает в личных и групповых чатах;
- для авто-анализа обычных сообщений в группе у бота должен быть отключён privacy mode в `@BotFather`;
- если авто-черновики выключены, бот работает только по явным командам;
- любые авто-черновики считаются предложением, а не окончательным решением.
- анализ переписки идёт по последним сообщениям обсуждения, а не по одному случайному сообщению.

### Основные модули
- `travel_planner.py` — парсинг свободного запроса и генерация черновика поездки
- `database.py` — хранилище поездок, участников, дат и chat settings с SQLite/Postgres
- `bot/handlers.py` — Telegram-сценарии и команды
- `bot/formatters.py` — текстовые “страницы” внутри Telegram: onboarding, help, summary, settings
- `bot/trip_service.py` — сборка и обновление поездки как предметного объекта

## Быстрый запуск
1. Создайте бота через `@BotFather`.
2. Подготовьте `.env` с переменными:
   - `TELEGRAM_BOT_TOKEN`
   - `DATABASE_URL` или `DATABASE_PATH`
   - `OPENROUTER_API_KEY` и `OPENROUTER_MODEL`, если нужен `/planai`
   - `OPENROUTER_WEB_SEARCH=true`, если хотите, чтобы LLM подтягивал более свежую travel-информацию через OpenRouter web search
3. Установите зависимости:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

4. Запустите:

```bash
python app.py
```

## Render deploy
В репозитории уже есть `render.yaml`. Для деплоя нужны:
- `TELEGRAM_BOT_TOKEN`
- `DATABASE_URL` для Postgres в production
- `OPENROUTER_API_KEY`, если используется `/planai`
- `OPENROUTER_MODEL=stepfun/step-3.5-flash:free`, если нужен быстрый бесплатный LLM-режим
- `OPENROUTER_WEB_SEARCH=true`, если нужен доступ модели к web search для более свежих travel-ответов
- `LOG_LEVEL`
- `PLAYWRIGHT_ENABLED=true`, только если позже захотите включить браузерный слой для `/hotels`
- `PLAYWRIGHT_TIMEOUT_MS`, опционально для Playwright-режима

При наличии `PORT` бот поднимает маленький health endpoint на `/healthz`.
