# Product Vision

## Product statement
Telegram-бот помогает группе друзей или коллег пройти путь от хаотичного обсуждения поездки к понятному общему плану внутри чата.

## Основной сценарий MVP
1. Пользователь запускает `/plan` или `/newtrip`, либо бот замечает обсуждение поездки в группе, если это разрешено в настройках.
2. Бот собирает черновик поездки:
   - направление;
   - длительность;
   - даты или сезон;
   - размер группы;
   - бюджетный ориентир;
   - интересы;
   - открытые вопросы.
3. Бот показывает summary как “одну точку правды” по активной поездке.
4. Участники отмечают статус участия и голосуют за даты.
5. Организатор обновляет детали короткими командами или edit-flow.

## Product map
### Core MVP
- создание поездки из свободного текста и через мастер;
- summary и travel-brief;
- маршрут, бюджет, логистика, проживание, альтернативы;
- статусы участников;
- голосование по датам;
- заметки и открытые вопросы;
- настройки чата;
- история поездок и повторная активация архивной поездки;
- help и onboarding внутри Telegram.

### Next
- более сильный AI-анализ переписки;
- правки плана естественным языком в одном сообщении;
- улучшенный sharing и подключение новых участников;
- история поездок и переключение между архивом и активной поездкой;
- реальные напоминания через job queue.

### Later
- live API цен на билеты и проживание;
- Telegram Mini App / web UI;
- бронирование;
- shared expenses и распределение бюджета по участникам;
- AI summary длинных обсуждений и action items.

## UX принципы
1. Telegram-first: все ключевые сценарии должны работать без внешнего интерфейса.
2. Один чат, одна активная поездка: бот должен ясно показывать, какой план сейчас активен.
3. Явность важнее магии: авто-анализ переписки включается и выключается в настройках чата.
4. Summary важнее деталей: маршрут и бюджет полезны, но центр продукта это общая сводка договорённостей.
5. Без ложных обещаний: если нет live API, бот называет бюджет ориентиром, а не точной ценой.

## Success criteria for MVP
- пользователь может создать и обновлять одну активную поездку в чате;
- группа может отметить статус участия и договориться по датам;
- summary достаточно, чтобы понять текущее состояние поездки без чтения всей переписки;
- настройки чата позволяют выключить авто-черновики и оставить только ручной режим;
- бот понятен новому пользователю через `/start` и `/help` без сопровождения разработчика.

## Public command surface
- `/start` — быстрый старт и навигация по сценарию;
- `/help` — возможности, ограничения, правила использования;
- `/plan` — создать поездку из свободного текста;
- `/newtrip` — создать поездку через мастер;
- `/summary`, `/brief` — показать текущее состояние поездки;
- `/trips`, `/select_trip` — посмотреть историю и выбрать активную поездку;
- `/itinerary`, `/budget`, `/route`, `/stay`, `/alternatives` — раскрыть детали;
- `/status`, `/participants`, `/adddate` — координация группы;
- `/settings` — управление режимом чата;
- `/archive_trip` — закрыть активную поездку без удаления истории;
