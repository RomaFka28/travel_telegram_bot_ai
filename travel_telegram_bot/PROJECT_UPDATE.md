# Обновление концепции продукта

## Было

Бот для организации поездок в групповом чате:
- собрать участников;
- согласовать даты;
- фиксировать бюджет;
- напоминать о незакрытых вопросах.

## Стало

AI travel-бот в Telegram, который:
1. понимает запрос на поездку обычным языком;
2. собирает travel-brief;
3. предлагает маршрут, бюджет и логистику;
4. затем помогает группе согласовать участие.

## Почему это изменение важно

Из заметок по продукту видно, что пользователю нужен не только координатор переписки, а помощник, который:
- подсказывает интересные места;
- учитывает сезонность;
- помогает понять бюджет;
- советует, как удобнее добраться;
- предлагает, где жить;
- умеет скорректировать направление и формат поездки.

## Новый MVP-фокус

### Слой 1. Планирование
- свободный текстовый запрос;
- структура поездки;
- маршрут по дням;
- бюджетный ориентир;
- логистика;
- рекомендации по проживанию;
- альтернативные направления.

### Слой 2. Координация группы
- кто едет / думает / не едет;
- голосование за даты;
- сводка;
- заметки и открытые вопросы.

## Что отложено

- live-подбор билетов;
- live-подбор отелей;
- автoчтение всей переписки;
- реальное бронирование;
- AI-резюме длинных чатов.

## Технический подход текущей версии

Текущая версия — это не LLM-интеграция, а хорошая рабочая база:
- Telegram-бот на `python-telegram-bot`;
- SQLite для хранения активных поездок;
- встроенный travel planner на эвристиках;
- чистые точки расширения под будущие API и LLM.

## Что нужно для следующего шага

1. Подключить LLM-слой для более гибкого понимания запросов.
2. Подключить travel API для live-цен.
3. Добавить web-view / mini app для удобного редактирования маршрута.
4. Включить контекстный анализ группового чата как отдельный режим.
